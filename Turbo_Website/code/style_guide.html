Introduction

Files
In general the whole Project consists of multiple modules which again contain several packages. The namespaces are equivalent to the module names and files in the module represent the packages. The Turbo files are separated by source and header files. Header files are in the include folder of the containing module and source files are in the source folder of the containing module.
  • Turbo:
    • include
      • APP
      • GUI
        • GUIObject.h
      • EXTERN
      source
      • APP
      • GUI
        • GUIObject.cpp
      • EXTERN
Naming

File names should have underscores, when the file name gets to complex:

Examples:

Good
abstract_schedule_manager.h

Bad
abstractScheduleManager.h

Comments

Indentation
Brackets of a control block are always aligned to each other and always at the next line after the definition of a function, class, ... .

Examples:

Good
highlightCode($code); ?>

Bad
highlightCode($code); ?>
