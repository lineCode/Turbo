Introduction

Files
Naming In general the whole Project consists of multiple modules which again contain several packages. The namespaces are equivalent to the module names and files in the module represent the packages. The Turbo files are separated by source and header files. Header files are in the include folder of the containing module and source files are in the source folder of the containing module.
  • Turbo:
    • include
      • APP
      • GUI
        • GUIObject.h
      • EXTERN
      source
      • APP
      • GUI
        • GUIObject.cpp
      • EXTERN

Naming
Type Documentation
Namespaces

upper case letter, separated by underscore

Examples: GUI
Classes and Structs

Classes and Structures represent the more complex data types. Turbo uses mixed case for naming these.

Examples: UIMenu, PathFinder, ResourceManager
Interfaces

Capital I and mixed case

Examples: IShape, IComparator
Enums

upper case letter, separated by underscore

Examples: FILE_STATE {OPEN, CLOSED, ...}
Iterators

i and underscore

Examples: i_file_buffer
global Constant

upper case letter, separated by underscore

Examples: FILE_PATH
Variables

name with underscore

Examples: file_counter
Member variables

m and underscore, separating word with underscore

Examples: m_file_size
Pointers

p and underscore

Examples: p_file_handle
local constants

c and underscore

Examples: c_class_counter
